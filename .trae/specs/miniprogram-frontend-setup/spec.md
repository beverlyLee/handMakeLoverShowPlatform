# 手工爱好者展示平台小程序前端 - Product Requirement Document

## Overview
- **Summary**: 搭建微信小程序前端开发环境，创建适配前后端分离架构的小程序工程，对接Flask后端API，配置路由、全局样式和必要依赖，为10个核心功能模块提供开发基础，视觉设计贴合手工DIY主题风格。
- **Purpose**: 建立完整的微信小程序前端开发框架，实现与现有Flask后端的无缝对接，提供标准的开发规范和基础设施，为后续功能开发奠定坚实基础。
- **Target Users**: 
  - 前端开发人员（进行功能开发）
  - 手工爱好者/客户（使用小程序浏览和购买手工作品）
  - 手工创作者/老师（管理作品和订单）

## Goals
- [Goal 1] 创建标准微信小程序工程结构，适配微信开发者工具
- [Goal 2] 实现前后端分离架构，完成与Flask后端的API对接
- [Goal 3] 配置完整的路由系统，支持10个核心功能模块的页面导航
- [Goal 4] 建立全局样式系统，贴合手工DIY主题风格（温暖、自然、手工艺感）
- [Goal 5] 封装通用工具库（请求、存储、状态管理等），提供开发基础
- [Goal 6] 为10个核心功能模块创建页面框架和基础模板

## Non-Goals (Out of Scope)
- 不实现具体的业务逻辑功能（只创建页面框架和基础架构）
- 不进行后端API的修改或新增
- 不实现微信登录的完整流程（只创建基础框架）
- 不包含完整的测试用例编写
- 不进行生产环境部署配置

## Background & Context
- 项目采用前后端分离架构，后端为Flask（Python），已具备10个API模块（auth, products, orders, users, messages, reviews, cart, favorites, search, main）
- 前端为微信小程序原生开发，目前`handicraft-miniprogram`目录仅包含基础文件，需要从零搭建工程
- 目标用户为手工创作者和手工爱好者，需要温暖、自然、有手工艺感的视觉设计
- 后端API遵循统一响应格式：`{code: 0, msg: "success", data: {}}`

## Functional Requirements

### 小程序工程架构
- **FR-1**: 创建标准微信小程序工程目录结构
  - 包含`pages`、`components`、`utils`、`assets`、`styles`等标准目录
  - 配置`app.js`、`app.json`、`app.wxss`三大入口文件

### 后端对接
- **FR-2**: 封装HTTP请求工具库
  - 统一封装`wx.request`，支持GET/POST/PUT/DELETE方法
  - 自动携带Token认证头
  - 统一处理响应格式（解析后端标准响应）
  - 统一错误处理和提示

- **FR-3**: 创建API配置和接口封装
  - 配置开发/生产环境的API基础URL
  - 按模块封装所有API接口（auth, products, orders等10个模块）
  - 支持请求超时、重试等基础配置

### 路由配置
- **FR-4**: 配置TabBar底部导航
  - 首页（作品展示）、分类/搜索、购物车/收藏、个人中心 四个核心Tab
  - Tab图标和文字风格贴合手工主题

- **FR-5**: 配置完整页面路由
  - 10个功能模块对应的页面路由注册
  - 支持页面参数传递和跳转

### 全局样式与主题
- **FR-6**: 建立全局样式系统
  - 定义主题色板（温暖色系：米色、棕色、暖橙等手工感配色）
  - 定义通用字体规范（标题、正文、辅助文字）
  - 定义间距、圆角、阴影等设计Token
  - 定义按钮、卡片、表单等通用组件样式

- **FR-7**: 创建通用组件库
  - 基础组件：导航栏、底部Tab、加载动画、空状态
  - 业务组件：作品卡片、用户头像、价格标签等

### 工具库与基础设施
- **FR-8**: 封装通用工具函数
  - 存储管理（Token、用户信息本地存储）
  - 日期格式化、价格格式化
  - 图片预览、文件上传基础封装
  - 防抖、节流等通用函数

- **FR-9**: 实现基础状态管理
  - 用户登录状态管理
  - 全局共享数据管理（简单方案，如app.globalData）

### 10个功能模块页面框架
- **FR-10**: 创建作品浏览模块页面框架
  - 首页（作品列表、轮播、分类入口）
  - 作品详情页

- **FR-11**: 创建搜索模块页面框架
  - 搜索页（搜索框、历史记录、热门搜索）
  - 搜索结果页

- **FR-12**: 创建用户认证模块页面框架
  - 登录授权页
  - 个人信息编辑页

- **FR-13**: 创建个人中心模块页面框架
  - 个人中心主页
  - 我的收藏页
  - 收货地址列表页

- **FR-14**: 创建订单管理模块页面框架
  - 订单列表页（按状态筛选）
  - 订单详情页
  - 创建订单页

- **FR-15**: 创建地址管理模块页面框架
  - 地址新增/编辑页
  - 地址选择页

- **FR-16**: 创建作品管理模块页面框架（老师端）
  - 我的作品列表页
  - 作品发布/编辑页

- **FR-17**: 创建老师订单管理模块页面框架
  - 老师订单列表页
  - 订单操作页

- **FR-18**: 创建角色切换模块页面框架
  - 角色切换页

- **FR-19**: 创建老师主页模块页面框架
  - 老师店铺主页
  - 粉丝/关注页

- **FR-20**: 创建消息通知模块页面框架
  - 消息列表页
  - 消息详情页

## Non-Functional Requirements

- **NFR-1**: 开发体验
  - 代码结构清晰，注释规范
  - 命名统一（遵循微信小程序开发规范）
  - 配置文件集中管理，易于修改

- **NFR-2**: 可维护性
  - 模块化封装，低耦合高内聚
  - API接口按模块组织，易于扩展
  - 样式变量化，主题切换便捷

- **NFR-3**: 兼容性
  - 适配微信开发者工具最新版
  - 支持基础库版本2.10.0+
  - 兼容不同屏幕尺寸（响应式布局）

- **NFR-4**: 性能
  - 页面启动时间优化（分包加载基础配置）
  - 请求封装支持缓存策略基础框架
  - 图片资源使用WebP格式基础配置

## Constraints

- **Technical**:
  - 必须使用微信小程序原生框架（WXML/WXSS/JS），不使用uni-app等第三方框架
  - 必须对接现有Flask后端API，不修改后端接口定义
  - 必须在`handicraft-miniprogram`目录下创建工程
  - 必须适配微信开发者工具

- **Business**:
  - 视觉风格必须贴合手工DIY主题
  - 必须支持10个核心功能模块的后续开发

- **Dependencies**:
  - 依赖微信开发者工具进行预览和调试
  - 依赖Flask后端服务进行API联调

## Assumptions

- 后端Flask服务运行在`http://127.0.0.1:5000`（开发环境）
- 用户已有微信小程序AppID或使用测试号
- 微信开发者工具已安装并配置好
- 后端API遵循README中定义的统一响应格式

## Acceptance Criteria

### AC-1: 小程序工程创建
- **Given**: 空的`handicraft-miniprogram`目录
- **When**: 使用微信开发者工具导入该目录
- **Then**: 项目能够正常识别和编译，无报错
- **Verification**: `programmatic`
- **Notes**: 检查是否有标准的小程序入口文件

### AC-2: API请求封装验证
- **Given**: 已配置好后端API地址
- **When**: 调用封装的请求工具访问`/api/health`接口
- **Then**: 能够正确获取后端返回的`{status: 'healthy'}`响应
- **Verification**: `programmatic`
- **Notes**: 需要后端服务运行在5000端口

### AC-3: TabBar导航配置
- **Given**: 小程序已启动
- **When**: 点击底部TabBar的各个图标
- **Then**: 能够正确切换到对应的页面
- **Verification**: `human-judgment`
- **Notes**: 检查图标和文字是否显示正常

### AC-4: 页面路由跳转
- **Given**: 在首页
- **When**: 点击作品卡片或使用API跳转
- **Then**: 能够正确跳转到作品详情页，并传递参数
- **Verification**: `programmatic`
- **Notes**: 使用小程序API进行页面跳转测试

### AC-5: 全局样式主题
- **Given**: 任意页面
- **When**: 查看页面视觉效果
- **Then**: 颜色、字体、间距符合手工主题风格（温暖、自然色系）
- **Verification**: `human-judgment`
- **Notes**: 检查是否有统一的设计Token和主题变量

### AC-6: 10个功能模块页面存在
- **Given**: 小程序工程结构
- **When**: 检查pages目录
- **Then**: 10个功能模块对应的页面目录和文件都已创建
- **Verification**: `programmatic`
- **Notes**: 页面包括但不限于：首页、作品详情、搜索、登录、个人中心、订单列表、订单详情、地址管理、作品管理、老师主页等

### AC-7: 通用组件可复用
- **Given**: 组件目录
- **When**: 在页面中引用通用组件
- **Then**: 组件能够正确渲染和工作
- **Verification**: `human-judgment`
- **Notes**: 测试至少3个通用组件（如卡片、按钮、加载）

### AC-8: 本地存储工具
- **Given**: 用户登录场景
- **When**: 调用存储工具保存Token和用户信息
- **Then**: 数据能够正确存入`wx.storage`并在后续访问中读取
- **Verification**: `programmatic`
- **Notes**: 使用小程序的存储API进行验证
